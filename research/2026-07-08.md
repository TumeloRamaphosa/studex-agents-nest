## 3. VOICEBOX (GitHub)

**What it is**
- Local-first, open-source AI voice studio (28K GitHub stars, MIT license).
- Voice cloning from seconds of audio. 7 TTS engines, 23 languages. Whisper-based dictation. Built-in REST API + MCP server.
- Desktop app (macOS/Windows/Linux). **Not** a server/enterprise tool.

**For Studex customer service?**
- Technically: Yes — it has an MCP server and REST API, so it could be hooked into an agent pipeline.
- Practically: No. It's a desktop app. To run it headless on the VM you'd need to run a full desktop environment or port-forwarding hacks. Not production-ready for call center use.
- **Better path for voice**: Use ElevenLabs (already have an API key in .env) via the existing `batch_text_to_audio` and `synthesize_speech` tools already available. That's your voice layer.

**Verdict**: Interesting project to watch. For Studex: not a Build Hour priority.
