## TOPIC 2: Agent Automation — Blotato Status + n8n Deploy Opportunity

### Blotato — Current State
- API confirmed working (TOOLS.md). 0 accounts connected — the pipeline is built but not activated.
- **Problem:** Blotato posts to social platforms. But Studex's bottleneck isn't posting — it's content creation and content strategy. Blotato without a content pipeline = nothing to post.
- **Fix required:** Connect Blotato to an AI content generation flow (e.g., n8n generates a week's captions → Blotato publishes them).

### n8n — Why It Matters More Than Blotato Right Now
- n8n is workflow automation with AI agent capabilities. Visual builder, Python/JS code support, HTTP requests, 400+ integrations.
- **n8n can automate:** Customer inquiry routing, order processing, content pipeline, lead qualification, Follow-up reminders, data collection from web.
- **2026 updates:** Multi-agent orchestration, native LLM nodes (Claude/GPT direct), memory integration, improved error handling.
- **Deploy target:** Orgo.ai VM (already running Ubuntu, 8GB RAM). One `curl` install command, runs as a service.
- **vs Make.com:** n8n is self-hosted = your data, no per-task cost. Make.com is cloud-only = lock-in + per-operation pricing. **n8n wins for Studex.**

### What to Build in n8n (Priority Order)
1. **Lead Qualification Flow:** Website form → n8n classifies lead (hot/warm/cold) → routes to WhatsApp or email
2. **Content Pipeline:** AI generates product caption → Blotato schedules post → reports back to n8n
3. **Order Processing:** Incoming order (Instagram DM / WhatsApp) → n8n creates order record → notifies Tumelo
4. **Follow-up Automation:** If no response in 48h → nudge message sent automatically

## TOPIC 3: Voicebox — Customer Service Voice Agent (Quick Read)

### What It Is
Voicebox is an **open-source AI voice studio** (40.6k GitHub stars, v0.5.0 April 2026). Think ElevenLabs + WisprFlow combined, runs locally on your machine.

**Core capabilities:**
- Voice cloning: 10 seconds of audio → clone any voice
- 7 TTS engines (Qwen3-TTS, Kokoro, Chatterbox, HumeAI TADA, etc.)
- 23 languages including Swahili, Arabic, Hindi
- Dictation: global hotkey → speak anywhere → transcript pastes into any text field
- MCP server built-in → can integrate with Claude Code, Cursor, OpenClaw agents
- Local-first: everything runs on your machine, no cloud dependency

### Studex Application
**Customer service voice agent** is the obvious use case:
- Clone Tumelo's voice → customers hear a familiar South African voice
- WhatsApp voice responses (text-to-voice replies)
- Agent can dictate order confirmations, delivery updates
- **Not needed right now** — Studex doesn't have the volume yet. Revisit when daily inquiries > 50.

**More immediately useful:** Voicebox's dictation for Tumelo — when drafting product descriptions, narrate instead of type. 3x speed improvement on content creation.

**Verdict:** File under "Phase 2." Current priority is landing paying customers, not polishing the voice UX.
