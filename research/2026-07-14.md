## TOPIC 2: Voicebox — Open-Source AI Voice Studio (GitHub: jamiepine/voicebox)

### Key Findings

- **What it is**: Local-first voice studio (41K GitHub stars, MIT license). Combines voice cloning + TTS + speech-to-text + dictation in one app. Free, runs entirely on your machine.
- **For Studex customer service**: Can clone a real human voice (CEO or agent) from a 10-second sample. Then any MCP-aware AI agent can SPEAK to customers in that cloned voice. The agent pipeline: customer message → AI processes → Voicebox responds in cloned South African voice.
- **Tech stack**: Tauri (Rust) desktop app + FastAPI backend + Qwen3 local LLM + Whisper STT. Runs locally (no cloud, no per-token costs).
- **Integration**: Ships with MCP server. One command: `voicebox.speak({text: "Your order is on the way", profile: "Robusca"})`. Any AI agent (Claude Code, Cursor, OpenClaw subagents) can call this.
- **7 TTS engines**: Qwen3-TTS, LuxTTS, Chatterbox, HumeAI TADA, Kokoro — multilingual (23 languages, includes Swahili). Quality rivals ElevenLabs at 0 cost.
- **Limitation**: Desktop app (macOS/Windows/Linux). Can't run headless on a server. But: the REST API runs on localhost. Could expose via VM's network for internal use.

### What This Means for Studex
- **Build a voice customer service agent**: Clone Tumelo's voice → connect to WhatsApp/website chat → AI responds in a warm, local voice. Differentiator no competitor has.
- **First step**: Install Voicebox on the Orgo VM or Mac Mini. Clone a voice profile. Test with a simple "order status" AI agent.
- **ROI**: Eliminates text-only chatbot experience. Customers hear Studex, not a robot.
