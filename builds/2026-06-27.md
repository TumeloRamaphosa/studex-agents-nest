## What Was Built

### 🛠️ Studex Workflow Engine
**Location:** `/workspace/studex-workflows/`
**Stack:** Node.js + Express (no heavy dependencies — ~81 npm packages)
**Purpose:** Webhook-based automation server that connects Studex's sales channels (Shopify, Facebook) to WhatsApp + Discord notifications

---

## Components

### 1. `server.js` — Workflow API Server
Express server running on port 3000 with these endpoints:

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Service health check |
| POST | `/webhooks/shopify/order` | Shopify order created webhook |
| POST | `/webhooks/facebook/lead` | Facebook lead form submission |
| POST | `/workflows/morning-ops` | Manual trigger: daily ops digest |

### 2. `workflows/shopify-order-alert.js`
When a Shopify order is created:
- Sends WhatsApp message to ops team with customer name, item count, total
- Posts Discord embed (green) with order details, payment status, fulfillment status
- Low stock items detected automatically

### 3. `workflows/facebook-lead-alert.js`
When a Facebook lead form is submitted:
- Sends WhatsApp alert with lead name, email, phone, message
- Posts Discord embed (blue) to #crm channel

### 4. `workflows/morning-ops-report.js`
Daily at 07:00 SAST (via cron):
- Pulls Shopify orders from last 24h
- Calculates revenue
- Posts Discord ops digest with revenue, top products, low stock alerts
- Sends WhatsApp summary to ops team

### ⚠️ VM Deployment: Requires Manual Steps
The Orgo VM currently has only the `.env` file transferred. Full deployment requires:

```bash
# On the VM (via Orgo dashboard terminal or SSH):
cd /workspace/studex-workflows

# Install dependencies
npm install

# Start with PM2 (auto-restarts on crash/reboot)
npm install -g pm2
pm2 start server.js --name studex-workflows
pm2 save
pm2 startup  # Enables restart on boot

# Verify
curl http://localhost:3000/health
```
