# Contributing to studex-naledi-content

Thank you for helping Naledi shine! 🎉

## Brand First

Before any code change, read `brand/voice.md`. All content generated by Naledi must adhere to the Studex Meat brand voice guidelines.

## Getting Started

```bash
git clone https://github.com/TumeloRamaphosa/studex-naledi-content.git
cd studex-naledi-content
npm install
cp .env.example .env
```

## Adding New Content Pillars

1. Document the new pillar in `brand/voice.md`
2. Add examples in `content/captions/examples/` (minimum 5)
3. Update `brand/approved-hashtags.md` with relevant tags
4. Update the content calendar template

## Content Quality Gates

All generated content must:
- ✅ Follow the brand voice guidelines
- ✅ Pass the banned-topics check (`npm run audit:content`)
- ✅ Include platform-appropriate hashtags (from approved list)
- ✅ Include at least one call-to-action

## Testing Content Generation

```bash
npm run generate:sample     # Generate one sample post
npm run generate:today       # Generate today's content pack
npm run audit:content        # Run content quality audit
```

## Pull Request Process

1. Branch: `content/`, `fix/`, `agent/`
2. Add generated examples for review
3. Update relevant docs
4. Tag the CMO or Brand Manager for content review

## Adding a New Campaign

1. Create `campaigns/[campaign-name]/` directory
2. Add campaign brief in `campaigns/[campaign-name]/brief.md`
3. Run `npm run generate:campaign -- --name=campaign-name`
4. Submit campaign pack for approval

## Questions

Open an issue or contact the Studex Marketing + CTO team.
